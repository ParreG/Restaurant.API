# restaurant-api

A REST API for managing restaurant bookings, tables, dishes and staff — built with ASP.NET Core and Entity Framework Core.

This was a solo backend project where I wanted to practice clean architecture with a proper service/repository pattern, JWT authentication with role-based access, and a booking engine that actually handles table availability logic.

---

## Tech Stack

- **ASP.NET Core** — Web API
- **Entity Framework Core** — ORM with SQL Server
- **JWT Bearer Auth** — Authentication and role-based authorization
- **Swagger / Swashbuckle** — API documentation

---

## Features

- **Booking management** — Create, update and delete reservations with real-time table availability checks
- **Table management** — Add tables, update positions and query available tables by time and guest count
- **Dish management** — Full CRUD for menu items, protected behind authorization
- **Guest management** — Look up guests by ID or email
- **Auth system** — Login, registration via invite codes, JWT tokens and SuperAdmin role protection
- **Clean architecture** — Controllers → Services → Repositories, with DTOs throughout

---

## Project Structure

```
├── Controllers        # API endpoints
├── Services           # Business logic (interfaces + implementations)
├── Repositories       # Data access layer (interfaces + implementations)
├── DTOs               # Data transfer objects for requests and responses
├── Models             # EF Core entities
├── Data               # DbContext
├── Configurations     # Extension methods for Program.cs (auth, services, swagger)
└── Migrations         # EF Core migrations
```

## Getting Started

### Prerequisites

- .NET 8 SDK
- SQL Server (or LocalDB for local development)

### Setup

1. Clone the repo

```bash
git clone https://github.com/ParreG/restaurant-api.git
cd restaurant-api
```

2. Add your settings to `appsettings.json` (never commit real secrets — use environment variables or user-secrets in production)

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "your-connection-string-here"
  },
  "Jwt": {
    "Key": "your-secret-key-here",
    "Issuer": "your-issuer",
    "Audience": "your-audience"
  }
}
```

3. Apply migrations

```bash
dotnet ef database update
```

4. Run the API

```bash
dotnet run
```

Swagger UI will be available at `https://localhost:{port}/swagger`

## Auth & Roles

| Role | Access |
|------|--------|
| SuperAdmin | Can create invite codes for new admins |
| Admin | Full access to bookings, tables, dishes and guests |
| Guest (unauthenticated) | Can create a booking and view the menu |

Registration requires a valid invite code — new invites can only be generated by a SuperAdmin.
